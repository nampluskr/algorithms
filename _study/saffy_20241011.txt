# 삼성 코딩테스트 B형 빈출 유형 - 최근 3년 기준 (대략 년 36회)
*** B형 시험 맨처음 나오는 테이블 복사

=====================================================================
[B형] DB 관리: 추가 / 삭제 / 수정 / *쿼리* (ID: 큰정수 / 문자열)
- 각 DB는 정적 배열로 전역변수로 지정 (문제에서 최대 개수 반드시 확인)
- 각 케이스 시작시 내용 초기화 (최적화 가능, MAX_DB -> dbCnt, memset)

[유형1] DB 부분검색: 해시테이블  + 우선순위큐 (연 5회 이상)

(예) 서점: 책 DB (제목 / *작가* / *카테고리* / 가격 / 재고 / 절판유무)
*** ID 해시테이블 활용
*** Main DB: 모든 정보 저장 (책 DB)
*** 분야별로 개별 리스트 관리 (인덱스 포함 필요 정보만 저장)
-> 작가별 DB: (이름 / 수입 )
-> 카테고리별 DB: 책 DB 인덱스 리스트 / 우선순위 큐 저장

*** 삭제 처리 방법: DB에서 직접 삭제 vs. 삭제유무 Flag 표시
*** Top-K 쿼리 (데이터 변경 유무 확인 / 삭제 유무 확인)
(Ex) 정렬 O(nlog n) vs. 우선순위큐 O(K log n)

-----------------------------------------------------------------------
[유형2] 1D 배열: 구간 쿼리 문제 / 2D 그리드 좌표 정보 그룹핑 (3~5회)
- 구간합 / 구간 최대 / 구간 최소 / 구간 중간값
- Sqrt Decomposition / Segment Tree

[유형3] 2D 그리드: 확산 / 최단경로 문제 (2~3회)

[유형4] 그래프/트리 - 최소비용 / 최대비용 경로 구하기 / LCA / 다익스트라 응용 (2~3회)

[유형5] 이진 탐색 / Parametric Search (2~3회)

-----------------------------------------------------------------------
[유형6] 패턴 해시테이블 저장: 1D 음원 / 2D 블록, 별자리 등 (2~3회)

[유형7] 문자열 파싱 (2~3회)

[유형8] 그룹 합치기 (리스트/트리, Union-Find) (2~3회)

[유형9] 부분 집합 최적화 (최대, 최소 조건 구하기)

-----------------------------------------------------------------------
** [B형 풀이 순서] 코딩부터 하면 안됨!
(1) 알고리즘 설계: O(n^2) -> O(n)-> O(log n) -> O(1)
(2) STL 이용 풀이 (테스트 케이스 통과) 
*** 최적화 옵션 끄고 테스트 할 것 (O2 옵션 미적용)
(3) 코드 최적화: 동적 메모리 -> 정적 메모리 변환 / 구조체 분리 / 함수 분리 -> 종료시까지 계속
*** 시험장 레퍼런스 코드: 퀵소드, 우선순위큐 코드 복사해서 수정

-----------------------------------------------------------------------
*** C++ 기준: 25 케이스 0.7~0.9초 이하 합격권 (실제 50 케이스 3초 미만 합격)
*** 시험장에서 코드 매뉴얼 및 레퍼런스 코드 참고 가능 (포기하지 말고 공부하고 나오자!)
*** [거의 안나옴] DP / 문자열 검색, 압축 / 위상정렬 / 프림, 크루스칼 / 벨만-포드, 플로이드-워셜 / Max Flow
*** 우선순위큐 / 해시 활용 문제 많이 풀 것
